Your codebase,
understood.

2context reads your git history and uses AI to extract architectural decisions, coding conventions, patterns, and design choices. The result is a structured knowledge base committed to your repo that your team and AI coding assistants can read directly.

$ npm install -g 2context
$ cd your-repo && 2context init

or run without installing: npx 2context init

// what it extracts

Four knowledge categories

Everything extracted is classified and stored as plain Markdown files you can commit alongside your code.

architecture

Structural decisions

Module boundaries, data flow, layer responsibilities, and major structural choices.

convention

Code style rules

API design choices, naming patterns, and team norms that live outside a linter.

decision

Technical choices

A specific decision and its rationale, captured at a point in time with context preserved.

pattern

Reusable solutions

Idioms and implementation patterns that appear repeatedly across the codebase.

// how it works

Self-maintaining by design

Knowledge is built incrementally, kept organized, and validated against the actual code.

incremental

Only processes what's new

State is tracked per source. Each run picks up from the last analyzed commit with no redundant work.

rebalancing

AI-powered clustering

Automatically splits overfull folders into subcategories and merges underfull ones as the graph grows.

validation

Staleness detection

Detects knowledge items whose source has changed or disappeared, and removes them.

agent-ready

Injects into CLAUDE.md

Automatically updates your CLAUDE.md or AGENTS.md so AI coding assistants pick up the full knowledge graph.

co-located

Lives next to your code

KNOWLEDGE.md files are placed alongside the code they describe, not hidden in a separate docs folder.

plain markdown

No lock-in

All output is plain Markdown committed to your repo. No external service, no proprietary format.
